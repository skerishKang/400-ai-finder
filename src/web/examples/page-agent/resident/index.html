이 화면은 무엇인가?

정밀 구현형 MVP(시민 행정 도우미)와 동일한 시민 화면에서, 모델 호환 plan을 통해 Page Agent가 실제 DOM 동작(클릭·입력·선택·스크롤·같은 출처 이동)을 수행하는 주민용 비교 데모입니다. 영어 개발자 문서(Page Agent 개발자 실험실)와는 별개의 제품 카드입니다.

공유 비교 시나리오 (5개)

정밀 구현형과 Page Agent형이 동일하게 시도하는 시나리오입니다. 정의는 examples/page-agent/parity-contract.json에 고정되어 있습니다.

  1. 공동주택과 연락처 찾아줘 — apartment_contact
  2. 대형폐기물 신청 메뉴 찾아줘 — bulky_waste_menu
  3. 여권 발급 절차를 찾아줘 — passport_procedure
  4. 민원 작성 화면을 열어줘 — complaint_screen
  5. 구청장에게 제안할 글 작성을 도와줘 — mayor_proposal_writing
제출 금지 경계: 두 모드 모두 실제 민원 제출·결제·로그인·개인정보 전송·외부 도메인 이동·임의 JavaScript 실행을 하지 않습니다. 화면 탐색·입력 보조·초안 제공까지만 수행하고 제출 전에 멈춥니다. 실제 북구청 사이트를 직접 조작하지 않습니다.

구현 상태

라우트 계약과 제품 라벨링, 공유 시나리오 계약은 Stage 1에서 확정되었습니다. Page Agent형 데모의 대화형 구현(실제 action loop 통과)은 Stage 2에서 진행됩니다. 이 페이지는 라우트 격리와 제품 라벨링 계약을 잠그기 위한 자리 표시자입니다.
