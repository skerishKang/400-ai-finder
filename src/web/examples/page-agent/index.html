Page Agent

An in-page GUI agent for web interaction — local interoperability experiment.

MIT License Offline Experiment alibaba/page-agent@1.12.1

What is Page Agent?

Page Agent is an open-source, in-page GUI agent that gives any web page its own AI assistant. It runs entirely in the browser as a JavaScript library — no Python runtime, no headless browser, no browser extension required. The agent can understand page content, navigate the DOM, fill forms, click buttons, and report results, all through a lightweight in-page integration.

Unlike traditional browser automation tools that require a separate runtime environment, Page Agent embeds directly into the page and operates with standard DOM APIs.

Core Features

  • Easy Integration — Add a single <script> tag to give any page an AI agent.
  • Text-based DOM Manipulation — Operates on the live DOM tree. No screenshots, no canvas capture, no special permissions.
  • Model Agnostic — Works with most mainstream LLMs and locally deployed models through an OpenAI-compatible API.
  • Built-in Panel UI — Ships with a ready-to-use floating chat panel (task input, status indicators, action history).
  • PageController — Integrated DOM interaction engine: click, type, select, scroll, and execute JavaScript on the page.
  • Chrome Extension — Optional companion extension for cross-page and multi-tab tasks.
  • MCP Server — Optional Model Context Protocol server for extended AI tool integration.

Use Cases

AI Copilot for SaaS

Embed an intelligent assistant directly into your web application to guide users through complex workflows, form filling, and data entry tasks.

Smart Form Filling

Automate repetitive form interactions — the agent can navigate multi-step forms, pre-fill known values, and validate input before submission.

Accessibility Enhancement

Provide an alternative interaction layer for users who benefit from natural-language commands instead of manual navigation.

Multi-page Browser Agent

Combine with the optional Chrome extension to perform tasks that span multiple pages or browser tabs.

MCP Client Control

Use the MCP server integration to connect the agent with external tools and data sources through the Model Context Protocol.

Page Agent vs browser-use

While browser-use is a Python framework for AI-powered browser automation using headless browsers, Page Agent takes a fundamentally different approach:

Aspect Page Agent browser-use
Runtime Browser (JavaScript) Python + headless browser
Integration In-page <script> tag Separate automation script
DOM Access Direct (same-origin) Via CDP/WebDriver
Multi-page Optional extension Built-in
Use Case Embedded AI copilot Programmatic automation

Quick Start

One-line integration using the published package:

<script src="https://cdn.jsdelivr.net/npm/page-agent@1.12.1/dist/iife/page-agent.demo.js" crossorigin="anonymous"></script>

This lab uses a local vendored copy of the same bundle with a deterministic mock model — no external CDN, no live API calls.

NPM Installation

Via npm for programmatic use:

npm install page-agent@1.12.1

Or save-exact for reproducible builds:

npm install --save-exact page-agent@1.12.1 --ignore-scripts

Basic Execution

Create an agent instance and execute a task:

import { PageAgent } from 'page-agent'

const agent = new PageAgent({
  model: 'qwen3.5-plus',
  baseURL: 'https://dashscope.aliyuncs.com/compatible-mode/v1',
  apiKey: 'YOUR_API_KEY',
  language: 'en-US',
})

await agent.execute('Click the login button')

In this lab, a custom local model adapter replaces the DashScope/OpenAI endpoint with a deterministic mock. The agent class, core, controller, and panel are the actual published implementations — only the model provider is substituted.

Architecture

Page Agent is composed of several layered modules:

Panel (UI) Floating chat interface — task input, status, history
PageAgent Orchestrator — extends PageAgentCore
PageAgentCore Agent loop — model interaction, planning, execution
LLM Client OpenAI-compatible HTTP client with custom fetch support
PageController DOM interaction engine — click, type, scroll, select
Custom UI Build your own interface on top of PageAgentCore

Custom UI

For applications that need a bespoke interface, PageAgentCore can be used directly without the built-in Panel. The core exposes agent lifecycle events (thinking, executing, completed, error) so you can build your own UI components:

import { PageAgentCore } from 'page-agent'

const core = new PageAgentCore({
  model: 'gpt-4o',
  baseURL: 'https://api.openai.com/v1',
  apiKey: 'sk-...',
})

core.on('status', (status) => { /* update UI */ })
core.on('activity', (activity) => { /* render step */ })

await core.execute('Fill the search form')

The built-in Panel (used in this lab) wraps PageAgentCore with a ready-to-use chat interface that includes task input, status indicators, and an expandable action history.
