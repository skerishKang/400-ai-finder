# 대표 대상 발표자료: AI파인더 사업화 제안

## 1. 제목

AI파인더: 복잡한 홈페이지를 사람 말로 찾아주는 AI 내비게이터 사업화 제안

## 2. 문제

기관, 대학, 공공기관, 기업 지원사업 홈페이지는 여전히 찾기 어렵다. 사용자는 정확한 메뉴명과 행정 용어를 알아야 원하는 정보를 찾을 수 있다.

이 문제는 단순 불편이 아니라 반복 문의, 신청 포기, 정보 접근 격차, 담당자 업무 부담으로 이어진다.

## 3. 고객

초기 고객은 홈페이지에 정보가 많고, 이용자가 자주 길을 잃는 조직이다.

- 공공기관
- 지자체
- 대학과 산학협력단
- 중소기업 지원기관
- 협회와 단체
- 교육기관
- 병원과 민원기관

## 4. 해결책

AI파인더는 사용자의 자연어 질문을 받아 홈페이지 구조, 게시판, 공지사항, 첨부문서를 분석하고 관련 페이지와 문서로 안내한다.

사용자는 “신청서 어디 있어?”, “지원사업 공고 어디서 봐?”, “제출서류 뭐야?”처럼 물어볼 수 있다.

## 5. 기존 챗봇과의 차이

기존 챗봇은 미리 등록된 FAQ 중심이다. AI파인더는 홈페이지 전체 구조와 문서를 기반으로 답변한다.

- FAQ가 없어도 홈페이지에서 찾는다.
- 공고문과 첨부파일을 함께 본다.
- 답변에 바로가기와 근거를 붙인다.
- 사용자의 다음 행동을 단계별로 안내한다.

## 6. MVP 범위

초기 MVP는 다음 기능만 구현한다.

1. 단일 URL 분석
2. 홈페이지 메뉴 구조 분석
3. 게시판과 첨부파일 수집
4. 문서 검색 인덱스 생성
5. 자연어 질문에 대한 근거 기반 답변
6. 관련 링크와 해야 할 일 제공

브라우저 확장프로그램, 자동 클릭, 자동 신청은 초기 범위에서 제외한다.

## 7. 수익모델

### 구축형

기관 홈페이지별 초기 구축비를 받는다.

### 구독형

월 단위 유지관리, 재수집, 답변 품질 점검, 관리자 리포트를 제공한다.

### 커스터마이징형

기관별 문서 양식, 보안 요구, 내부망 환경, 다국어 안내 등을 별도 개발한다.

## 8. 도입 패키지 예시

| 패키지 | 내용 |
|---|---|
| PoC | 홈페이지 1개, 공개 페이지 기반, 핵심 메뉴와 공고 분석 |
| Standard | 주요 게시판, 첨부문서, 검색 UI, 관리자 검토 기능 |
| Enterprise | 내부망, 로컬 모델, 기관별 위젯, 운영 리포트 |

## 9. 경쟁우위

AI파인더의 경쟁우위는 기술보다 문제 정의에 있다. 많은 기관은 챗봇을 원한다고 말하지만 실제 문제는 홈페이지 탐색 실패다.

AI파인더는 “대화형 FAQ”가 아니라 “홈페이지 길찾기”에 집중한다.

## 10. 실행 로드맵

- 1단계: 단일 URL 분석기
- 2단계: 홈페이지 지도 생성기
- 3단계: 문서 인덱스와 검색
- 4단계: 근거 기반 질의응답
- 5단계: 웹 UI
- 6단계: 기관 PoC 제안

## 11. 대표 관점의 핵심 판단

이 사업은 기존 AI 교육, 기관 컨설팅, 홈페이지 개선 수요, 공공기관 디지털 전환 수요와 연결된다.

초기에는 작은 PoC로 시작하고, 실제 기관 홈페이지 하나를 잘 처리하는 사례를 만드는 것이 중요하다.
