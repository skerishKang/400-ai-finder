# 시스템 구조

AI파인더는 홈페이지를 수집하고, 문서를 분석하고, 사용자의 질문에 맞는 근거 기반 답변을 생성하는 구조로 설계한다.

## 기본 흐름

```text
홈페이지 URL
  ↓
수집기
  ↓
문서 파서
  ↓
메뉴 구조 분석기
  ↓
검색 인덱스
  ↓
질의응답 엔진
  ↓
출처 포함 답변
```

## 주요 구성요소

### 1. 홈페이지 수집기

- 시작 URL 수집
- sitemap.xml 확인
- 내부 링크 탐색
- 게시판 후보 탐지
- 첨부파일 링크 수집

### 2. 문서 파서

- HTML 본문 추출
- PDF 텍스트 추출
- HWP/HWPX 처리 계획 수립
- DOCX, XLSX 처리
- 표, 목록, 링크 정보 보존

### 3. 메뉴 구조 분석기

- 메인 메뉴와 하위 메뉴 추출
- URL 간 연결 관계 정리
- 사용자가 이해하기 쉬운 사이트 지도 생성

### 4. 검색 인덱스

- 키워드 검색
- 의미 기반 검색
- 문서 메타데이터 검색
- URL과 첨부파일 관계 검색

### 5. 질의응답 엔진

- 사용자 질문 의도 파악
- 관련 문서 검색
- 근거 기반 답변 생성
- 출처 링크와 문서명 표시

## 모델 연결 방식

모델은 고정하지 않는다. 초기 구조는 다음 방식을 모두 허용한다.

- API 모델
- 로컬 모델
- API 모델과 로컬 모델의 하이브리드

모델보다 중요한 것은 답변이 반드시 수집된 문서와 URL 근거에 기반해야 한다는 점이다.
