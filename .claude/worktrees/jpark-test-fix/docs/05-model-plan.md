# 모델 활용 계획

## 기본 방향

AI파인더는 특정 모델에 종속되지 않는다. 초기 MVP는 모델을 쉽게 교체할 수 있는 구조로 설계한다.

## 선택 가능한 모델 구성

### 1. API 모델

장점:

- 빠른 MVP 구현
- 높은 답변 품질
- 문서 요약과 질의응답에 유리

단점:

- 외부 API 비용 발생
- 기관 내부 문서 처리 시 보안 검토 필요

### 2. 로컬 모델

장점:

- 외부 API 의존도 감소
- 내부망 또는 폐쇄망 납품 가능성
- 기관 문서 처리 시 보안 메시지 강화

단점:

- 초기 세팅과 성능 튜닝 필요
- 긴 문서 처리와 한국어 품질 검증 필요
- 하드웨어 요구사항 발생

### 3. 하이브리드 모델

장점:

- 공개 홈페이지 문서는 API 모델 사용
- 민감한 내부 문서는 로컬 모델 사용
- 비용과 품질을 균형 있게 조정 가능

## 핵심 원칙

모델은 답변을 생성하는 역할을 하지만, 사실의 출처는 반드시 수집된 홈페이지와 문서여야 한다.

## 답변 원칙

- 검색된 문서와 URL에 근거해 답한다.
- 문서명, URL, 게시일, 첨부파일명을 함께 표시한다.
- 신청기한, 금액, 제출서류, 담당자 정보는 원문 근거가 있을 때만 확정 표현한다.
- 불확실한 정보는 확인 필요로 표시한다.
- 근거가 없으면 모른다고 답한다.
