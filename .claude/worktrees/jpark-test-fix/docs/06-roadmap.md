# 로드맵

## Stage 0. 프로젝트 초기화

목표:

- README 작성
- 기본 문서 작성
- 데이터와 모델 파일 제외 설정
- 초기 폴더 구조 생성

결과물:

- README.md
- docs/
- prompts/
- examples/
- data/
- src/

## Stage 1. 단일 URL 분석

목표:

- 사용자가 URL 하나를 입력하면 제목, 본문, 링크를 추출한다.

예상 결과:

```json
{
  "url": "https://example.com",
  "title": "페이지 제목",
  "text": "본문 텍스트",
  "links": []
}
```

## Stage 2. 홈페이지 지도 생성

목표:

- 메인 페이지와 sitemap.xml을 기반으로 주요 메뉴 구조를 정리한다.

예상 결과:

```text
홈페이지
├─ 공지사항
├─ 사업공고
├─ 자료실
└─ 신청안내
```

## Stage 3. 문서 수집과 색인

목표:

- 수집된 HTML과 첨부파일을 검색 가능한 문서 단위로 저장한다.
- 초기에는 JSONL 기반 색인을 사용한다.

## Stage 4. 근거 기반 질의응답

목표:

- 사용자 질문에 대해 관련 문서를 먼저 찾고, 그 근거만 사용해 답변한다.

## Stage 5. 간단한 웹 UI

목표:

- 홈페이지 지도, 질문창, 답변, 관련 링크를 한 화면에서 제공한다.

## Stage 6. 확장 기능 검토

후보:

- 브라우저 확장프로그램
- 기관 홈페이지 삽입형 위젯
- 관리자용 수집 상태 대시보드
- 로그인 없는 공개 페이지 실시간 분석
- 기관별 맞춤 검색 인덱스
