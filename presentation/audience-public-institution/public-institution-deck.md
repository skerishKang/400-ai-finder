# 공기업 및 기관 대상 발표자료: AI파인더 도입 제안

## 1. 제목

AI파인더: 기관 홈페이지 이용 편의성 향상을 위한 지능형 탐색·안내 시스템

## 2. 기관 홈페이지의 현실적 문제

기관 홈페이지는 정보량이 많고 메뉴 구조가 복잡하다. 이용자는 정확한 행정 용어와 메뉴명을 알지 못하면 원하는 정보를 찾기 어렵다.

정보는 여러 위치에 분산되어 있다.

- 공지사항
- 사업공고
- 자료실
- 신청안내
- 첨부문서
- 문의 안내

## 3. 이용자 관점의 불편

이용자는 홈페이지에서 다음과 같은 어려움을 겪는다.

- 어디로 들어가야 하는지 모른다.
- 공지와 사업공고의 차이를 모른다.
- 신청서가 본문이 아니라 첨부파일 안에 있다.
- 제출서류와 기한을 한눈에 확인하기 어렵다.
- 모바일 화면에서 메뉴 탐색이 더 어렵다.

## 4. 담당자 관점의 부담

홈페이지에서 정보를 찾지 못한 이용자는 담당자에게 반복적으로 문의한다.

반복 문의는 담당자의 업무 시간을 줄이고, 이용자는 답변을 기다려야 한다. 기관 입장에서는 이미 공개한 정보가 제대로 활용되지 못하는 문제가 생긴다.

## 5. AI파인더 개요

AI파인더는 기존 홈페이지를 대체하지 않는다. 기존 홈페이지 위에 AI 탐색 레이어를 추가해 이용자가 원하는 정보를 더 쉽게 찾게 한다.

사용자는 자연어로 질문한다.

- 신청서 어디 있어?
- 이 사업 제출서류가 뭐야?
- 지원사업 공고 어디서 봐?
- 교육 신청은 어디로 들어가야 해?

AI파인더는 관련 페이지, 문서, 절차, 링크를 찾아 안내한다.

## 6. 주요 기능

1. 홈페이지 메뉴 구조 분석
2. 공지사항과 사업공고 탐색
3. 자료실과 첨부문서 분석
4. 사용자의 자연어 질문 이해
5. 관련 페이지와 문서 추천
6. 신청 절차와 제출서류 요약
7. 답변 근거와 원문 링크 제공

## 7. 도입 방식

초기 도입은 기존 홈페이지 개편 없이 가능하다.

- 공개 홈페이지 기반 수집
- 주요 메뉴와 공고 분석
- 기관 전용 검색 인덱스 구성
- 이용자용 질문창 제공
- 관리자 검토 후 답변 품질 개선

장기적으로는 홈페이지 삽입형 위젯 또는 내부망 기반 시스템으로 확장할 수 있다.

## 8. 보안과 개인정보 원칙

초기 버전은 공개 홈페이지와 공개 문서만 대상으로 한다.

- 로그인 정보 수집 없음
- 개인정보 입력값 수집 없음
- 자동 신청 또는 자동 제출 없음
- 답변은 공개 문서 근거 기반으로 제한

기관 요구에 따라 로컬 모델 또는 내부망 운영도 검토할 수 있다.

## 9. 근거 기반 답변 구조

AI파인더는 단순히 답변만 생성하지 않는다. 답변과 함께 근거를 제공한다.

답변에는 다음 항목을 포함한다.

- 관련 페이지명
- URL
- 문서명
- 게시일
- 첨부파일명
- 확인 필요 사항

근거가 없는 정보는 확정적으로 답하지 않는다.

## 10. 기대효과

### 이용자 측면

- 원하는 정보를 더 빨리 찾는다.
- 신청 절차를 쉽게 이해한다.
- 첨부문서 안의 핵심 정보를 빠르게 확인한다.
- 모바일과 PC에서 동일한 안내를 받을 수 있다.

### 기관 측면

- 반복 문의를 줄인다.
- 홈페이지에 공개된 정보의 활용도를 높인다.
- 담당자 안내 부담을 줄인다.
- 이용자 만족도를 높인다.
- 정보 접근성과 디지털 포용성을 강화한다.

## 12. 운영·유지관리

홈페이지는 계속 바뀌므로 정기적인 재수집과 검토가 필요하다.

운영 항목:

- 정기 수집
- 변경 페이지 감지
- 답변 품질 점검
- 오류 신고 반영
- 관리자 리포트 제공
