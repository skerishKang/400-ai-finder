# 문제 정의

## 사용자가 겪는 문제

많은 홈페이지는 사용자의 목적보다 기관 내부 조직과 행정 용어를 기준으로 구성되어 있다. 그래서 사용자는 정확한 메뉴명, 사업명, 문서명을 알지 못하면 원하는 정보를 찾기 어렵다.

## 대표적인 어려움

1. 메뉴명이 직관적이지 않다.
2. 같은 정보가 여러 게시판과 자료실에 흩어져 있다.
3. 핵심 정보가 본문이 아니라 별도 문서 파일 안에 있는 경우가 많다.
4. 신청 페이지와 안내 페이지가 분리되어 있다.
5. 제출서류, 신청기한, 담당자, 대상자 정보가 한눈에 보이지 않는다.
6. 홈페이지 검색창은 사용자의 생활 언어를 잘 이해하지 못한다.
7. 모바일 화면에서는 메뉴가 더 숨겨져 탐색이 어렵다.

## AI파인더가 풀어야 할 핵심 질문

- 사용자가 무엇을 찾고 싶은가?
- 그 정보는 홈페이지의 어느 메뉴와 문서에 있는가?
- 사용자가 지금 바로 눌러야 할 링크는 무엇인가?
- 사용자가 해야 할 다음 행동은 무엇인가?
- 답변의 근거가 되는 원문, 문서, 파일은 무엇인가?
