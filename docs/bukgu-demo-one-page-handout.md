# 400-ai-finder 데모 요약자료

## 특정 기관 사이트를 AI와 대화하면서 탐색하는 출처 기반 정보검색 도구

### 1. 문제의식

공공기관 홈페이지에는 공지사항, 고시공고, 민원, 복지, 청년·일자리 정보가 여러 메뉴와 게시판에 흩어져 있습니다.

사용자는 필요한 정보를 찾기 위해 여러 메뉴를 직접 이동해야 하고, 기관마다 메뉴 구조와 게시판 명칭도 달라 검색이 쉽지 않습니다.

일반 검색엔진이나 챗봇은 답변이 빠르더라도, 실제 공식 사이트에 근거했는지 확인하기 어렵다는 한계가 있습니다.

400-ai-finder는 특정 공공기관 사이트를 실제로 읽고, 사용자의 질문에 맞춰 공식 출처 기반 답변을 생성하는 AI 탐색 도구입니다.

---

### 2. 400-ai-finder가 하는 일

400-ai-finder는 특정 사이트를 대상으로 사용자의 질문을 받아, 관련 페이지를 찾고, 실제 LLM이 그 자료를 바탕으로 답변을 생성합니다.

기본 흐름은 다음과 같습니다.

1. 사용자가 특정 기관 사이트에 대해 질문합니다.
2. 시스템이 해당 사이트의 관련 페이지를 실시간으로 가져옵니다.
3. LLM이 가져온 자료를 바탕으로 답변을 생성합니다.
4. 답변과 함께 출처 URL을 확인할 수 있습니다.
5. 공식 도메인에 근거했는지 검증할 수 있습니다.

핵심은 AI가 일반 지식을 추측하는 것이 아니라, 실제 사이트에서 가져온 자료에 근거해 답변한다는 점입니다.

---

### 3. 북구청 데모 시나리오

이번 데모의 대상은 광주광역시 북구청 사이트입니다.

| 항목 | 내용 |
|---|---|
| 대상 사이트 | 광주광역시 북구청 |
| 도메인 | `bukgu.gwangju.kr` |
| site profile | `bukgu_gwangju` |
| 검증 LLM provider | NVIDIA |
| 검증 모델 | `openai/gpt-oss-120b` |
| fetch provider | `requests` |
| Firecrawl | 미사용 |

데모 질문은 다음 3개를 중심으로 진행합니다.

1. 북구청에서 최근 공지사항이나 고시공고를 어디에서 확인할 수 있나요?
2. 북구청 민원 신청이나 온라인 민원 안내는 어디에서 확인할 수 있나요?
3. 북구청 사이트에서 청년 또는 일자리 관련 정보를 찾으려면 어떻게 접근해야 하나요?

각 질문에 대해 답변 내용뿐 아니라, 답변 하단의 출처 URL이 북구청 공식 도메인 또는 관련 공식 출처에 근거하는지 함께 확인합니다.

---

### 4. 실제 검증 결과

400-ai-finder는 mock/offline 테스트를 넘어 실제 live fetch와 실제 LLM을 사용한 제품 검증을 완료했습니다.

| 검증 항목 | 결과 |
|---|---|
| 공지사항·고시공고 질문 | PASS |
| 복지 지원사업 질문 | PASS |
| 민원 신청·온라인 민원 질문 | PASS |
| 청년·일자리 질문 | PASS |
| 출처 도메인 검증 | WARN, 별도 검색질문이 아니라 답변 출처 확인 방식이 적절 |

북구청 관련 질문에 대해 실제 사이트를 fetch하고, LLM이 그 자료를 바탕으로 답변을 생성하는 end-to-end 흐름이 정상 작동했습니다.

---

### 5. 데모에서 강조할 포인트

첫째, AI가 단순히 기억이나 일반 지식으로 답변하지 않습니다.
실제 사이트에서 가져온 자료를 바탕으로 답변합니다.

둘째, 답변의 출처를 확인할 수 있습니다.
답변이 북구청 공식 도메인이나 관련 공식 출처에 근거하는지 볼 수 있습니다.

셋째, 특정 기관별 정보 탐색으로 확장할 수 있습니다.
북구청뿐 아니라 다른 지자체, 공공기관, 복지기관, 경찰·행정기관 사이트로 확장할 수 있습니다.

### 6. 기대 활용처

- 지자체 홈페이지 민원·복지·공지 안내
- 공공기관 내부 정보 탐색
- 고시공고·지원사업·교육사업 검색
- 시민 대상 북구청 AI 민원 네비게이터
- 특정 기관 사이트 기반 RAG형 챗봇
- 여러 기관 사이트를 대상으로 한 AI 통합 안내 서비스

---

### 7. 현재 단계

현재 400-ai-finder는 다음 단계를 완료했습니다.

- mock/offline 검증 완료
- live fetch + live LLM 검증 완료
- 북구청 대상 실제 제품 검증 완료
- 데모 리허설 완료
- 외부 시연용 데모 패키지 작성 완료

현재 상태는 단순 MVP를 넘어, 제한된 조건에서 외부 시연 가능한 controlled demo 단계입니다.

---

### 8. 한 줄 설명

400-ai-finder는 특정 공공기관 사이트를 AI가 직접 읽고, 사용자의 질문에 대해 공식 출처 기반 답변을 생성하는 사이트 대화형 정보탐색 도구입니다.
